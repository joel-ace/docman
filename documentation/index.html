    Introduction

    DocMan is a document management system API that allows easy CRUD operations for documents. Users can create, edit, retrieve and delete documents. It also offers a way to ensure that only authorized users can perform certain operations.

    Development

    The application was developed with NodeJs and Express for routing. Postgres is the database used and sequelize is the ORM used.

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of Node.js equal or greater than v6.8.0.

    1. Clone the repository from a terminal git clone https://github.com/joel-ace/docman.git.
    2. Change into the docman directory cd docman
    3. Rename .env_sample to .env and add the required DATABASE URL.
    4. Install project dependencies npm install
    5. Start the express server npm start.

    Authentication

    All routes are protected and needs a valid token (there's an exception in the login and signup). To get a token, the user has to lgin with the email and password used in creating the account. A token is included in the response from the login route, this token expires after 24 hours. With a valid token a user can create/update/delete or search for documents, the request must contain the valid token on the header of the request in the following format.

    Authorization: token

    You must replace token with your personal token.

    API Summary

    Users

    EndPoint Functionality
    POST /api/v1/users/login Logs in a user.
    POST /api/v1/users/ Creates a new user.
    GET /api/v1/users/ Gets all users.
    GET /api/v1/users/:id Gets user by id.
    PUT /api/v1/users/:id Updates a user's attributes based on the id specified (available only to the account owner)
    DELETE /api/v1/users/:id Deletes user (available to the account owner and admin)
    GET /api/v1/users/:id/documents Gets all documents for a particular user

    Documents

    EndPoint Functionality
    POST /api/v1/documents/ Creates a new document.
    GET /api/v1/documents/ Gets all documents.
    GET /api/v1/documents/:id Find document by id.
    PUT /api/v1/documents/:id Updates a document attributes. (available only to the document owner)
    DELETE /documents/:id Delete document. (available only to the document owner and admin)

    Search

    EndPoint Functionality
    GET /api/v1/search/users/?q=${query} Gets all users with fullname or email containing the search query
    GET /api/v1/search/documents/?q=${query} Gets all documents with title containing the search query

    Errors

    If you are new to HTTP status codes, you can use this as a reference to know what the returned error status codes used in DocMan means.

    DocMan API uses the following code error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your API key is wrong or you do not have sufficient privilege.
    404 Not Found -- What you requested for could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily unavailable. Please try again later.
